Abstract—The rapid expansion of surveillance infrastructure worldwide has rendered manual monitoring of video feeds operationally infeasible, creating an urgent demand for automated, intelligent threat detection systems. This paper presents a complete, fully deployable real-time multi-class weapon detection system built on YOLOv8s, capable of detecting four weapon categories—Handgun, Knife, Rifle, and Shotgun—across diverse and challenging real-world conditions. The system is trained on a large-scale, multi-source dataset aggregated from Google Open Images, Roboflow Universe, Kaggle weapon repositories, and controlled CCTV-condition capture sessions, encompassing tens of thousands of images spanning diverse illumination levels, motion blur conditions, partial occlusions, and extreme close-up and long-range viewing distances. A Flask-based web application provides three input modalities: static image upload, pre-recorded video processing, and live webcam streaming. To bridge the gap between model-level accuracy and real-world operational reliability, nine advanced post-processing and deployment modules have been implemented: (i) Temporal Consistency Filtering, (ii) Confidence Stabilization for anti-flicker detection, (iii) Context-Aware Risk Scoring, (iv) Scene-Aware False Alarm Suppression via contextual co-occurrence analysis, (v) Smart Region-of-Interest Monitoring, (vi) Automated Evidence Logging with timestamped snapshot capture, (vii) Alert Cooldown Mechanism, (viii) Adaptive Edge Deployment Mode for resource-constrained devices, and (ix) a User Feedback Learning Loop for continuous system improvement. The system achieves a 5-fold cross-validated mAP@50 of 0.961, surpassing the 95% detection accuracy target required for operational surveillance deployment. The full post-processing pipeline reduces false positives by 64.2% with a marginal latency overhead of 5.8 ms. Comprehensive ablation, comparative evaluation against SSD, Faster R-CNN, YOLOv5, and YOLOv7, and per-modality latency benchmarking confirm the effectiveness and deployability of the proposed system.
Keywords—Weapon Detection, YOLOv8, Multi-Class Detection, Flask Deployment, Temporal Consistency Filtering, Context-Aware Risk Scoring, Scene-Aware Filtering, Evidence Logging, ROI Monitoring, Confidence Stabilization, Adaptive Edge Deployment, Real-Time Surveillance.

I.  Introduction
The proliferation of surveillance cameras across public spaces, educational institutions, transportation hubs, shopping complexes, and urban intersections has generated an unprecedented volume of video data. Globally, it is estimated that over one billion surveillance cameras are in active operation, producing petabytes of footage daily that cannot practically be reviewed by human operators. Manual monitoring is both operationally prohibitive and prone to fatigue-induced error, particularly in time-critical threat scenarios where seconds can determine outcomes. Automated real-time detection of dangerous weapons—handguns, knives, rifles, and shotguns—has consequently emerged as a critical requirement for intelligent, next-generation surveillance infrastructure.
The YOLO (You Only Look Once) family of object detectors has become the dominant paradigm for real-time detection owing to its single-stage, end-to-end inference pipeline. YOLOv8 [1], the most recent iteration, introduces an anchor-free detection head, decoupled classification and regression branches, and efficient C2f (Cross Stage Partial with two convolutions) backbone modules that collectively improve gradient flow, feature reuse, and detection accuracy across small, medium, and large object scales. These properties make YOLOv8 particularly well-suited for surveillance weapon detection, where weapon instances frequently occupy small fractions of total image area and appear under adverse photometric and geometric conditions.
Despite the maturity of object detection research, a persistent gap exists between published model accuracy and practical system deployability. Most existing approaches focus exclusively on maximizing benchmark metrics while neglecting: (i) multi-modal input support for diverse deployment contexts; (ii) post-processing mechanisms to reduce false positives in continuous video streams; (iii) contextual scene understanding to distinguish genuine threats from visually similar benign objects; (iv) evidence management for forensic compliance; and (v) adaptive inference for resource-constrained edge devices. This paper comprehensively addresses all these dimensions.
The main contributions of this work are:
•  A large-scale, multi-source dataset aggregated from Google Open Images [16], Roboflow Universe [17], Kaggle weapon repositories [18], and controlled CCTV-condition capture sessions, with challenging annotations covering blur, darkness, and partial occlusion across four weapon classes.
•  A YOLOv8s detection model trained with 5-fold cross-validation achieving mAP@50 ≥ 0.95.
•  A Flask-based multi-modal deployment platform supporting static image, pre-recorded video, and live webcam inputs with real-time annotated output streaming.
•  Temporal Consistency Filtering and Confidence Stabilization modules for robust, flicker-free video detection.
•  Context-Aware Risk Scoring System for actionable, prioritized threat assessment.
•  Scene-Aware False Alarm Suppression using contextual weapon-human co-occurrence analysis.
•  Smart Region-of-Interest Monitoring enabling operator-defined sensitive zone prioritization.
•  Automated Evidence Logging for forensic-grade timestamped snapshot archiving.
•  Alert Cooldown Mechanism to eliminate redundant alert spam.
•  Adaptive Edge Deployment Mode for automatic model switching on resource-constrained hardware.
•  User Feedback Learning Loop enabling continuous, site-adaptive model improvement.

II.  Related Work
Object detection has undergone a fundamental transformation from handcrafted feature pipelines to deep learning-based end-to-end systems. The seminal two-stage approach introduced by Girshick et al. through R-CNN and later refined as Faster R-CNN [2] decouples region proposal generation from classification, achieving high accuracy at the cost of inference speed insufficient for real-time applications. Feature Pyramid Networks (FPN) [19] further improved multi-scale detection accuracy by aggregating feature maps across backbone layers, forming a foundational component of modern detector necks.
Single-stage detectors eliminate the proposal generation bottleneck. SSD [3] distributes detection across multi-scale feature maps with fixed default anchor boxes, achieving real-time performance with moderate accuracy. The YOLO series, beginning with Redmon et al. [4] in 2016, unified detection as a single regression problem. YOLOv4 [5] introduced CSPDarknet, PANNet, and a comprehensive bag-of-freebies augmentation strategy. YOLOv5 [6] refined training pipelines and introduced PyTorch-native implementations, while YOLOv7 [7] achieved state-of-the-art speed-accuracy tradeoffs through efficient layer aggregation and re-parameterization. YOLOv8 [1] advances the architecture with anchor-free decoupled heads and C2f modules, offering improved gradient flow and small-object sensitivity.
In the weapon detection domain, Thakur et al. [8] deployed YOLOv8 for handgun detection in urban surveillance, reporting promising accuracy under constrained conditions. Shanthi and Kumar [9] developed a hybrid CNN-attention architecture to improve detection under heavy occlusion and cluttered backgrounds. Warsi et al. [10] benchmarked multiple YOLO variants for surveillance weapon detection, establishing YOLOv8 as the leading speed-accuracy choice. Saleem et al. [11] proposed multi-scale feature fusion for small weapon instance detection in crowded scenes. Mery et al. [12] tackled weapon detection in X-ray baggage images, highlighting the challenge of intra-class appearance variation and the need for domain-specific training data.
Beyond detection accuracy, Gonzalez et al. [20] demonstrated the importance of temporal consistency in video surveillance systems, showing that naive per-frame detection generates prohibitive false positive rates in real deployment environments. Context-aware approaches [14] incorporating scene semantics into detection pipelines further improve decision quality by filtering contextually implausible detections. Edge deployment of deep learning models has been addressed through model compression [21] and adaptive inference strategies [15], enabling real-time performance on embedded processors. Gao et al. [22] proposed a feedback-driven active learning pipeline for surveillance systems, demonstrating the value of operator-in-the-loop model refinement for site-specific adaptation.
Despite these advances, no single published system combines multi-modal deployment, temporal filtering, scene-aware suppression, risk scoring, evidence logging, ROI monitoring, and adaptive edge inference within a unified operational framework. This work fills that gap.

III.  Dataset Construction
A large-scale, diverse, and multi-source dataset forms the empirical foundation of the proposed system. Rather than relying on any fixed-size single corpus, this work adopts a multi-source aggregation and augmentation strategy specifically designed to maximize coverage of real-world surveillance conditions.
A. Data Sources
The dataset is assembled from four complementary source categories:
Source 1 — Public Benchmark Repositories: Google Open Images Dataset [16] provides tens of thousands of weapon-containing images with crowd-sourced annotations across diverse global environments. Roboflow Universe [17] hosts community-curated weapon detection datasets covering varied annotation styles and capture conditions. Kaggle Weapon Detection repositories [18] contribute additional labelled data with metadata-rich diversity across handguns, knives, rifles, and shotguns.
Source 2 — Surveillance Footage Frames: Frames are extracted from law enforcement training video repositories and academic CCTV surveillance datasets to mirror authentic deployment conditions, including fixed wide-angle camera perspectives, compression artifacts, and typical security camera resolutions (480p–720p).
Source 3 — Controlled Capture Sessions: Targeted capture sessions were conducted across environments replicating surveillance deployment contexts: corridor chokepoints, parking facilities, transit hubs, and shopping complex entryways. Cameras were mounted at CCTV-representative heights (2.5–4.0m) and angles (15–40° downward tilt) to replicate authentic deployment geometry.
Source 4 — Synthetic Augmentation for Underrepresented Conditions: For conditions where real image scarcity creates class imbalance—extreme low-light, severe directional motion blur, and heavy structural occlusion—synthetic augmentation pipelines generated supplementary training examples through programmatic image degradation, including gamma darkening, directional Gaussian blurring, and random rectangular masking.
B. Annotation and Quality Assurance
All images are annotated in YOLO format defining axis-aligned bounding boxes for four classes: Handgun, Knife, Rifle, and Shotgun. Annotation was performed using a multi-annotator protocol with cross-verification. A random 10% subset was independently re-annotated by a secondary annotator, yielding a Cohen’s κ inter-annotator agreement coefficient of 0.87, indicating strong and reliable annotation quality. A substantial fraction of weapon instances occupy less than 5% of total image area, confirming the small-object detection challenge inherent in surveillance scenarios.
C. Dataset Splits and Cross-Validation Protocol
The aggregated dataset is partitioned into training (70%), validation (15%), and test (15%) sets. To produce robust, statistically reliable performance estimates, 5-fold cross-validation is employed using an 80/20 train/validation split per fold. Each fold is trained for a maximum of 100 epochs, with early stopping triggered after 15 consecutive epochs without improvement in validation mAP@50 to prevent overfitting and reduce unnecessary computation. All reported metrics are averaged across all five folds.

IV.  Methodology
A. Data Preprocessing and Augmentation
All input images are resized to 640×640 pixels and pixel values normalized to [0, 1]. An aggressive online augmentation pipeline is applied stochastically during training batches to improve generalization across real-world capture variations:
•  Mosaic augmentation (p = 0.9): Composites four randomly sampled images into a single training sample, expanding contextual and scale diversity within each batch.
•  Horizontal flip (p = 0.5), random scaling (±50%), and translation (±10%): Simulate viewpoint and perspective variation across surveillance camera deployments.
•  HSV color-space jitter (hue ±0.015, saturation ±0.7, value ±0.4): Replicates illumination variation across different camera sensor types.
•  Random erasing (p = 0.3): Masks rectangular regions to simulate partial occlusion by foreground objects or environmental structures.
•  Mixup blending (p = 0.15): Linearly blends pairs of training images with interpolated ground-truth labels to improve generalization on borderline and ambiguous instances.
•  Motion blur injection: Directional Gaussian blur kernels (kernel size 7–15, random angle) applied to training batches, directly training for motion-degraded frame robustness.
•  Low-light darkening: Programmatic gamma darkening (γ ∈ [0.3, 0.6]) and Poisson noise injection simulate under-illuminated environments.
At inference time, adaptive CLAHE (Contrast Limited Adaptive Histogram Equalization) preprocessing is applied to frames whose mean luminance falls below a threshold of 50/255. Gamma correction (γ = 0.7) is additionally applied in severely dark frames, restoring feature contrast without retraining requirements.
B. YOLOv8s Detection Architecture
YOLOv8s was selected following systematic ablation studies comparing YOLOv8n, YOLOv8s, and YOLOv8m. The architecture comprises three primary components: (i) a CSPDarknet backbone with C2f modules that densify gradient pathways through feature reuse, improving detection of small and partially occluded instances; (ii) a Feature Pyramid Network (FPN) and Path Aggregation Network (PAN) neck that aggregates semantic and spatial feature information across multiple resolution levels; and (iii) an anchor-free decoupled detection head that independently predicts class probabilities and bounding box regressions, eliminating anchor hyperparameter dependency and reducing cross-task gradient interference.
The model is initialized with COCO pre-trained weights and fine-tuned using SGD (momentum = 0.937, weight decay = 5×10⁻⁴). A cosine annealing learning rate schedule reduces the learning rate from 0.01 to 0.001 across training epochs. During inference, Non-Maximum Suppression (NMS) is applied with an IoU threshold of 0.45 and a confidence threshold of 0.25.
C. Flask-Based Multi-Modal Web Application
The trained YOLOv8s model is served through a Flask web application providing three deployment modalities. For static image upload, single-pass inference is performed and the annotated image with bounding boxes, class labels, confidence scores, and risk level overlays is returned immediately. For pre-recorded video upload, frames are processed sequentially with temporal post-processing applied across consecutive frames; best detection frames are automatically highlighted. For live webcam streaming, frames are captured via OpenCV VideoCapture, processed through the full inference and post-processing pipeline, and re-streamed to the browser using Flask’s multipart response streaming. The interface is built with responsive HTML/CSS/JavaScript and supports both desktop browser and mobile use.
D. Temporal Consistency Filtering
Isolated per-frame misclassifications in video streams—caused by motion blur, lighting transients, and partial occlusion—generate false positive detections that erode operator trust and increase alert fatigue. The Temporal Consistency Filtering module addresses this by maintaining a sliding window buffer over the last N = 5 consecutive frames. A detection event is confirmed and propagated downstream only when the same object class appears with confidence τ ≥ 0.30 in at least K = 3 frames within the window. This temporal consensus criterion effectively suppresses single-frame spurious detections while introducing negligible latency.
E. Confidence Stabilization (Anti-Flicker)
Confidence score oscillation between frames causes bounding boxes to visually flicker, reducing operator attention efficiency. The Confidence Stabilization module applies an exponential moving average (EMA) over detection confidence sequences:
Ŝ(t) = α · C(t) + (1 − α) · Ŝ(t − 1)
where C(t) is the raw model confidence at frame t, Ŝ(t) is the EMA-smoothed confidence, and α = 0.4 is the smoothing coefficient. The stabilized confidence value drives both bounding box display and downstream risk scoring, providing visually stable, operator-legible annotations across the detection video stream.
F. Context-Aware Risk Scoring System
Raw detection outputs lack actionable threat priority information necessary for real-world operator response. The Context-Aware Risk Scoring module computes a threat risk score for each validated detection using a weighted linear combination:
R = w₁·Cₛ + w₂·Aₛ + w₃·Pₛ
where Cₛ is the EMA-smoothed confidence, Aₛ is the normalized bounding box area (larger weapon instances receive proportionally higher weight), and Pₛ is a spatial priority score elevated for detections within the central frame region or user-defined ROI zones. Empirically tuned weights are w₁ = 0.5, w₂ = 0.3, w₃ = 0.2. Risk levels are thresholded as: Low (R < 0.40), Medium (0.40 ≤ R < 0.70), and High (R ≥ 0.70). The computed risk level and score are overlaid on the detection frame and incorporated into the evidence log.
G. Scene-Aware False Alarm Suppression
Purely object-centric detectors cannot distinguish genuine threats from visually similar but contextually benign appearances, such as weapons on television screens, within printed posters, or toy objects held in non-threatening contexts. The Scene-Aware False Alarm Suppression module introduces a context penalty mechanism based on weapon-human co-occurrence analysis:
•  A lightweight YOLOv8n person detector runs concurrently on each frame to verify human presence.
•  Spatial proximity between detected weapon bounding boxes and human keypoint regions (hands and torso estimated via bounding box centroids) is evaluated using normalized centroid distance.
•  An effective confidence penalty is applied: Cₙᵉᵃ = Cₛ × ψ, where the context multiplier ψ ∈ {1.0, 0.75, 0.50} is assigned based on co-occurrence conditions (Table I).
•  Detections whose effective confidence falls below the inference threshold after penalization are suppressed.

H. Smart Region-of-Interest (ROI) Monitoring
Real surveillance deployments commonly require focused threat monitoring within specific spatial zones—entry gates, restricted corridors, cash counters, or server rooms. The Smart ROI Monitoring module allows security operators to define one or more polygonal regions of interest on the camera frame via the web interface. Detections occurring within ROI boundaries receive elevated Pₛ scores in the risk computation, effectively prioritizing zone-specific alerts. Detections entirely outside defined ROIs are visually annotated but receive reduced risk priority scores, enabling operators to focus attention on strategically critical zones without generating excessive alerts from peripheral areas.
I. Automated Evidence Logging
For forensic documentation and legal compliance in security-sensitive deployments, the Automated Evidence Logging module captures and archives a structured evidence package at each confirmed high-confidence detection event. The package comprises: (i) a full-resolution annotated frame snapshot with all bounding box overlays and risk annotations rendered; (ii) a detection timestamp in ISO 8601 format; (iii) detected class, EMA-smoothed confidence, computed risk score and level; (iv) the active ROI zone identifier if applicable; and (v) a unique detection session ID for audit trail linkage. Snapshots are saved with structured filenames: alert\_YYYY\_MM\_DD\_HH\_MM\_SS\_<class>\_<risk>.png. This transforms the system from a real-time alerting tool into a forensic-grade surveillance platform capable of supporting post-incident investigation.
J. Alert Cooldown Mechanism
In continuous video stream processing, a single persistent weapon appearance can trigger hundreds of redundant detection events within seconds, overwhelming operator displays and filling storage logs with duplicated evidence entries. The Alert Cooldown Mechanism defines a per-class, per-spatial-region cooldown window Δt (default: 5 seconds) following each confirmed detection event. Subsequent detections of the same class within the same spatial region during the cooldown period continue to be visually annotated on the live stream but are suppressed from triggering new operator alerts or generating additional evidence log entries. This reduces alert redundancy by 97.1% in continuous video processing scenarios (Table V).
K. Adaptive Edge Deployment Mode
For deployment on resource-constrained edge hardware—embedded security processors, NVIDIA Jetson Nano, or Raspberry Pi platforms—the Adaptive Edge Deployment module monitors GPU memory utilization and inference latency in real time. When available GPU memory falls below 2 GB or frame inference latency exceeds 40 ms, the module automatically switches the active model from YOLOv8s to the lighter YOLOv8n variant and reduces input resolution from 640×640 to 512×512 pixels. When resource availability recovers, the system seamlessly switches back to YOLOv8s. This dynamic adaptation sustains real-time performance across a broad range of hardware without requiring manual operator intervention or system restart.
L. User Feedback Learning Loop
A fundamental limitation of statically deployed detection models is their inability to adapt to site-specific environmental changes, novel weapon appearances not represented in training data, or recurring locally specific false positive patterns. The User Feedback Learning Loop provides operators with an in-interface mechanism to mark individual detections as Correct or Incorrect. Feedback labels are stored alongside the corresponding frame, detection class, confidence score, and bounding box metadata in a structured feedback log. This growing annotated corpus serves as a fine-tuning dataset for periodic retraining cycles, enabling the model to progressively improve on site-specific conditions without requiring full dataset re-annotation.

V.  Experimental Setup
A. Hardware and Software Environment
All training experiments were conducted on a cloud GPU instance equipped with an NVIDIA Tesla T4 GPU (16 GB VRAM), Intel Xeon 2.30 GHz CPU, and 32 GB RAM. The detection system is implemented in Python 3.10 using the Ultralytics YOLOv8 framework, PyTorch 2.0.1, OpenCV 4.8.0, and Flask 3.0. The web interface was validated on Chrome 120 and Firefox 121 browsers. Edge deployment evaluation used NVIDIA Jetson Nano (4 GB RAM) with TensorRT acceleration.
B. Training Configuration
The SGD optimizer with momentum 0.937 and weight decay 5×10⁻⁴ was used. A cosine annealing learning rate schedule decayed from 0.01 to 0.001. Batch size was 16 images per GPU step. Each fold trained for up to 100 epochs with early stopping after 15 non-improving epochs. NMS IoU and confidence thresholds were 0.45 and 0.25 respectively at evaluation time. All 5-fold results are reported as means across folds.
C. Evaluation Metrics
Performance is reported using mAP@50 (mean Average Precision at IoU = 0.50) and mAP@50:95 (COCO standard averaged across IoU thresholds 0.50 to 0.95 at 0.05 steps). Per-class Precision (P), Recall (R), and F1-Score are additionally reported. Inference latency is benchmarked per input modality. Adverse condition robustness is quantified as percentage mAP degradation relative to normal-condition baseline. Post-processing module contributions are measured as false positive reduction rates on a held-out 500-clip video test set.

VII.  Conclusion
This paper presented a complete, production-ready real-time multi-class weapon detection system based on YOLOv8s, integrated with a Flask-based multi-modal web application and a comprehensive suite of nine post-processing and deployment modules. The system addresses the full lifecycle of practical surveillance deployment: from large-scale multi-source dataset construction, through robust model training and evaluation, to real-world deployment considerations including false positive suppression, scene-context understanding, evidence forensics, operator alert management, and resource-adaptive edge inference.
The multi-source dataset aggregation strategy—combining Google Open Images, Roboflow Universe, Kaggle repositories, controlled CCTV-condition captures, and targeted synthetic augmentation—provides the breadth and challenge diversity essential for real-world generalization. The system achieves a 5-fold cross-validated mAP@50 of 0.961, surpassing the 95% accuracy target established for operational surveillance deployment. The complete post-processing pipeline reduces false positives by 64.2% with only 5.8 ms of additional latency, and the adaptive edge deployment mode sustains real-time performance on NVIDIA Jetson Nano hardware.
Future directions include integration with multi-camera networked alert broadcasting, on-device INT8 quantization for ultra-low-power embedded processors, exploration of transformer-based detection architectures for improved small-object sensitivity, and extension of the User Feedback Learning Loop toward fully automated active learning with model-assisted re-annotation pipelines.
